### C++ Day 4
#### 1. 도형 편집기 예제
* 모든 도형을 타입으로 설계하자
* 모든 도형의 기반 클래스(Shape)가 있으면 모든 도형을 하나의 컨테이너로 관리할 수 있음
    * std::vector<Shape*>
* 모든 파생 클래스(도형)의 공통의 특징이 있다면 반드시 기반 클래스(Shape)에도 있어야 함
    * 문법적 규칙이 아니라, 디자인 규칙!
    * 그래야 Shape*로 모든 도형을 묶어서 관리할 때 해당 기능을 사용할 수 있음
* 기반 클래스가 가진 멤버 함수 중에서 파생 클래스가 재정의(override)하는 것은 반드시 가상함수 이어야 함
* 다형성(Polymorphism): 동일한 표현식(함수 호출)인데, 상황(실제 객체)에 따라 다르게 동작
* 객체지향 언어의 3대 특징: 캡슐화, 상속, 다형성

#### 2. 추상 클래스
* 추상 클래스: 순수 가상 함수가 한 개 이상 있는 클래스
    * 순수 가상함수 (pure virtual function): 구현부가 없고, =0 으로 표기
        * 가상함수: 파생 클래스 반드시 재정의 할 필요 없음. 재정의 안하면 기본 구현 제공
        * 순수 가상함수: 반드시 특정함수를 만들라고 지시
* 특징: 객체를 생성할 수 없음
* 의도: 파생 클래스에게 특정 함수를 반드시 만들라고 지시하는 것
* 추상 클래스를 상속 받은 클래스는 순수 가상함수를 구현하지 않으면 추상 클래스
    * 순수 가상함수를 구현하면 추상 클래스 아님

#### 3. 인터페이스
* 강한 결합 (tightly coupling)
    * 하나의 클래스가 다른 클래스 사용 시 이름을 직접 사용하는 것
    * 교체 불가능한 경직된 디자인. 확장성 없음
* 약한 결합 (loosely coupling)
    * 하나의 클래스가 다른 클래스 사용 시 규칙을 담은 추상 클래스를 통해서 사용하는 것
    * 교체 가능한 유연한 디자인, 확장성 있음

|인터페이스|추상 클래스|
|---------|----------|
|지켜야 하는 규칙(순수 가상 함수)만 있는 경우|지켜야 하는 규칙(순수 가상 함수) + 다른 멤버|

* C++은 인터페이스에 대한 별도 키워드 없음
* "모든 카메라는 아래 인터페이스를 구현해야 한다" 라고 표현
* 인터페이스를 만들 때 struct를 사용하는 경우도 많음 (public을 빼도 됨)
* 인터페이스도 결국 기반 클래스. 소멸자에 virtual 꼭 붙이기
* virtual은 함수 선언부에만 붙여야 함. 구현부에 붙이면 에러.
