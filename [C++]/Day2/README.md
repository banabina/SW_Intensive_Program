5. 초기화리스트
    => 생성자 () 뒤에 : 을 적고 멤버 데이터를 초기화하는 것
    => 초기화 시에는 초기화 리스트를 사용하는 것이 좋은 코드!
    => user define type은 항상 초기화가 빠름
    => 초기화 리스트는 코드가 놓인 순서가 아니라 멤버가 선언된 순서로 초기화됨
6. 정적멤버
    static 멤버 데이터
      => 정적 멤버는 모든 객체가 공유함
      => static 멤버 데이터는 클래스 외부에서 반드시 정의 되어야 함
         객체를 한개도 생성하지 않아도 static 멤버 데이터는 메모리에 생성됨
    static 멤버 함수
      => 객체 없이 호출 가능
    static 멤버 함수에서는 static 멤버만 접근 가능

핵심 정리
1. 필요한 타입을 먼저 만들자

3. 생성자로 객체가 자동으로 초기화 될 수 있게 하자.

주의
 - 클래스 템플릿을 사용할 때는 반드시 타입을 전달해야 함
 - 생성자는 2개 이상 만들 수 있음. 모든 객체를 만들 때 생성자가 호출됨
 - 생성자를 한 개도 만들지 않으면 컴파일러가 인자 없는 생성자(디폴트 생성자) 제공
