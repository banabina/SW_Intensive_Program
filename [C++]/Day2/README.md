#### 2. OOP
* 필요한 타입을 먼저 만들자
* 타입을 만들 때 상태를 나타내는 데이터와 상태를 가지고 연산을 수행하는 함수를 묶어서 만들자!
* 객체(object): 세상에 존재하는 모든 것

#### 3. 접근 지정자
* private, public을 사용해서 안전하게 사용할 수 있도록 하자
* 멤버 데이터를 private에 놓아서 외부에 잘못된 사용으로 부터 객체의 상태가 불안해지는 것을 막음
      * 객체의 상태는 잘 정의된 멤버 함수를 통해서만 변경할 수 있음
* 접근 지정자를 생략하면 struct: public, class: private

#### 4. 생성자
* 생성자로 객체가 자동으로 초기화 될 수 있게 하자
* 클래스 이름과 동일한이름의 함수. 반환 타입 표기 하지 않음
* 생성자는 2개 이상 만들 수 있음. 모든 객체를 만들 때 생성자가 호출됨
* 객체를 생성하면 자동으로 호출
* 생성자를 한 개도 만들지 않으면 컴파일러가 인자 없는 생성자(디폴트 생성자) 제공
* 소멸자 : "~클래스이름" 모양의 함수
      * 반환값과 인자가 없다. 오직 한개만 만들수 있다. 객체가 파괴 될때 호출.
      
#### 5. 초기화리스트
*  생성자 () 뒤에 : 을 적고 멤버 데이터를 초기화하는 것
*  초기화 시에는 초기화 리스트를 사용하는 것이 좋은 코드!
*  user define type은 항상 초기화가 빠름
*  초기화 리스트는 코드가 놓인 순서가 아니라 멤버가 선언된 순서로 초기화됨


#### 6. 정적멤버
* static 멤버 데이터
     * 정적 멤버는 모든 객체가 공유함
     * static 멤버 데이터는 클래스 외부에서 반드시 정의 되어야 함
     * 객체를 한개도 생성하지 않아도 static 멤버 데이터는 메모리에 생성됨
* static 멤버 함수
     * 객체 없이 호출 가능
     * static 멤버 함수에서는 static 멤버만 접근 가능
